什么是Stream

  • stream不是集合元素，不是数据结构，并不保存数据，更像是一个高级版本的iterator
  • stream会隐式的在内部进行遍历，会做出我们需要的数据转换操作
  • 就像一个迭代器，单向， 不可往复，数据只能遍历一次
  • 可以并行化操作，利用Fork/Join

流的构成

使用一个流，通常有如下的步骤：

  • 获取一个数据源
  • 对数据进行转换
  • 执行操作获取想要的结果

每次转换原油Stream对象不改变，返回一个新的Stream对象，可以进行多次转换，就像是一条链条一样进行排列，变成一个管道，管道中进行处理操作

  • 从Collection和数组

    • JarFile.stream()

流的操作类型分为两种

  • Intermediate

    一个流可以跟零个或者多个intermediate操作，母的主要是打开流，然后做某种程度的数据映射/过滤，然后返回一个新的流。调用这类方法，并没有真正开始进行流的遍历

  • Terminal

    一个流只能有一个terminal操作，当这个操作执行，流就真正开始进行遍历

  • short-circuiting

    • 对于一个 intermediate 操作，如果它接受的是一个无限大（infinite/unbounded）的 Stream，但返回一个有限的新 Stream。
    • 对于一个 terminal 操作，如果它接受的是一个无限大的 Stream，但能在有限的时间计算出结果。

每次执行Intermediate操作，每次对Stream的元素进行转换，时间复杂度并不是n * O（N），因为转换操作是lazy的，多个转换操作只会在Terminal操作的时候融合起来，一次循环完成，可以这样理解，Stream里有个操作函数的集合，每次转换操作就是把转换函数放入这个集合中，在Terminal操作的时候循环Stream的集合，然后执行

流的构造

提供三种包装类型的Stream：

  • IntStream
  • LongStream
  • DoubleStream

常见的操作:

  • Intermediate

    map (mapToInt, flatMap 等)、 filter、 distinct、 sorted、 peek、 limit、 skip、 parallel、 sequential、 unordered

  • Terminal

    forEach、 forEachOrdered、 toArray、 reduce、 collect、 min、 max、 count、 anyMatch、 allMatch、 noneMatch、 findFirst、 findAny、 iterator

  • Short-circuiting

    anyMatch、 allMatch、 noneMatch、 findFirst、 findAny、 limit
